# SecureAI Gateway - Hong Kong PCPD AI Framework Mapping

## Purpose of this mapping

This document helps enterprise teams discuss how SecureAI Gateway capabilities can be mapped to Hong Kong AI governance review themes during an enterprise evaluation or focused pilot.

## Scope and disclaimer

This document is provided for enterprise evaluation and focused pilot discussions only. It does not represent legal advice, regulatory approval, certification, endorsement or confirmation of compliance by PCPD or any regulator.

## 1. AI Strategy and Governance

### Framework focus

Organisations should define how AI is approved, governed, monitored and reviewed across business teams.

### How SecureAI Gateway supports this area

SecureAI Gateway is designed to support AI access governance, approved-model access, user / department / model / use case visibility and policy-based routing.

### Evidence available during evaluation

- Tenant policy configuration
- Approved model routing configuration
- User, department, model and use case usage views
- Trace ID records for governed requests

### Customer responsibilities / not covered by the product

Customers remain responsible for AI governance policy ownership, legal assessment, business approval, training, operating procedures and regulator-facing submissions.

## 2. Risk Assessment and Human Oversight

### Framework focus

Organisations should assess AI risks and maintain appropriate human oversight for higher-risk use cases.

### How SecureAI Gateway supports this area

SecureAI Gateway can apply prompt inspection, sensitive-data masking / blocking, policy decisions and human review trigger patterns before requests reach approved AI models.

### Evidence available during evaluation

- Policy match records
- Mask / block / allow / review decisions
- Trace ID and audit evidence
- Request context including user, department, model and use case

### Customer responsibilities / not covered by the product

Customers define risk appetite, escalation rules, human review procedures and final decision accountability.

## 3. Customisation, Implementation and Management of AI Systems

### Framework focus

Organisations should manage how AI systems are configured, deployed and monitored.

### How SecureAI Gateway supports this area

SecureAI Gateway supports managed Control Plane configuration, model routing, policy enforcement and a customer-controlled Data Plane option for selected enterprise deployment patterns.

### Evidence available during evaluation

- Control Plane policy and route settings
- Data Plane deployment model notes
- Audit evidence for configuration and routing decisions
- Usage visibility by tenant, user, department, model and use case

### Customer responsibilities / not covered by the product

Customers are responsible for enterprise architecture approval, network design, identity integration, third-party model-provider selection and production operating controls.

## 4. Communication and Engagement with Stakeholders

### Framework focus

Organisations should explain AI governance controls and accountability to relevant stakeholders.

### How SecureAI Gateway supports this area

SecureAI Gateway provides governance evidence, request-level Trace ID and audit views that can support internal security review, compliance review and stakeholder discussions.

### Evidence available during evaluation

- Evaluation screenshots
- Audit evidence examples
- Deployment model notes
- Framework-aligned discussion material

### Customer responsibilities / not covered by the product

Customers decide what to disclose to staff, customers, suppliers, auditors and regulators, and remain responsible for formal notices and legal communications.

## Summary

SecureAI Gateway is framework-aligned with AI access governance, approved-model access, prompt inspection, sensitive-data masking / blocking, human review trigger patterns, audit evidence and customer-controlled Data Plane options. It supports enterprise review but does not replace legal, compliance or regulatory assessment.
